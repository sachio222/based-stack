## Two-Pass Workflow for Marketing

For marketing site work, use a two-pass approach:

**Pass 1: Load Knowledge Frameworks** that shape how you think about the problem -- StoryBrand for messaging structure, Cialdini for persuasion, Hormozi for offer design, etc.

Load `product-marketing-context` at the start of a project to ground everything in the product's specific positioning, audience, and voice.

## Subagent Types

When Claude uses the Agent tool, it can spawn specialized subagents. Each type has different capabilities.

| Type | What It Does | When to Use | Can Edit Files? |
|---|---|---|---|
| `developer` | Implements code with tests. Full tool access. | Building features, writing code, fixing bugs. The workhorse. | Yes |
| `Explore` | Fast codebase exploration. Searches files, reads code, answers questions about the repo. | Finding files, understanding patterns, "how does X work?" | No |
| `Plan` | Designs implementation approaches. Identifies critical files, considers tradeoffs. | Planning before building. "How should we approach X?" | No |
| `researcher` | Explores codebases and domains. Explains landscape, options, best practices. Read-only. | Learning about unfamiliar territory before planning. | No |
| `architect` | Understands architecture, project conventions, quality design patterns. | Architecture decisions, evaluating design options. | Yes |
| `debugger` | Systematic evidence gathering for complex bugs. | Hard-to-find bugs where the cause isn't obvious. | Yes |
| `quality-reviewer` | Reviews code and plans for production risks, project conformance, structural quality. | Code review before merging. Catches issues a human reviewer would flag. | Yes |
| `technical-writer` | Creates documentation optimized for LLM consumption. | Writing docs that Claude will read in future sessions. | Yes |
| `content-developer` | Writes and edits content following project style guides and templates. | Content chunks: articles, docs, marketing copy. | Yes |

**Subagent features:**
- **Worktree isolation**: `isolation: "worktree"` gives a subagent its own copy of the repo on a separate branch. Useful for parallel work that might conflict.
- **Background execution**: `run_in_background: true` for work you don't need immediately. You get notified when it finishes.
- **Parallel launch**: Send multiple Agent tool calls in a single message to run subagents concurrently.

## Common Orchestration Patterns

### Build Something Complex
```
1. Explore agent -> understand the codebase
2. /planner -> design the approach
3. /deepthink -> analyze key decisions
4. /decision-critic -> stress-test decisions
5. developer subagents (parallel, worktrees) -> build components
6. developer subagent -> integrate
7. /incoherence -> consistency check
8. preview tools -> visual verification
9. /problem-analysis -> if anything breaks
```

### Debug a Problem
```
1. /problem-analysis -> systematic investigation
2. Explore agent -> find relevant code
3. debugger subagent -> fix
4. developer subagent -> implement fix
5. preview tools -> verify
```

### Write Content
```
1. /product-marketing-context -> load context, set voice
2. content-developer subagent -> write the content
3. /incoherence -> cross-reference consistency
```

### CRO Audit (Per Page or Full Site)
```
Pass 1 (Frameworks): Load evaluation lens
1. /cro-methodology -> audit framework, hypothesis design, prioritization
2. /made-to-stick -> SUCCESs checklist (Simple, Unexpected, Concrete, Credible, Emotional, Stories)
3. /influence-psychology -> Cialdini's 6 principles (reciprocity, commitment, social proof, authority, liking, scarcity)

Pass 2 (Execution): Audit each page
4. /product-marketing-context -> load personas, differentiators, customer language, objections
5. /page-cro -> run full CRO checklist against each page

Output: Per-page findings with specific recommendations, prioritized by impact.
Run this on any new marketing page before publishing.
```

### Make a Decision
```
1. researcher subagent -> gather context
2. /deepthink -> structured analysis
3. /decision-critic -> adversarial review
4. Save decision rationale to a doc
```

## Tips

- **Skills invoke immediately.** Don't analyze first, then invoke. The skill orchestrates its own exploration.
- **Subagents start fresh.** Each Agent invocation gets no prior context. Give it a complete task description with all the files it needs to read.
- **Parallel when independent, sequential when dependent.** If two subagents don't need each other's output, launch them in parallel.
- **Worktrees for parallel code changes.** If two subagents would edit the same files, give them worktrees. Merge after.
- **Background for non-blocking work.** If you're waiting on research while building something else, run the research agent in the background.
- **Quality skills after building, not during.** Run `/incoherence`, `/simplify` after the work is done, not while writing.
